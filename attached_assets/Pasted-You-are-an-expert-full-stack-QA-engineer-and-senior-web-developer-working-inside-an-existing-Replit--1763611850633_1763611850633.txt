You are an expert full-stack QA engineer and senior web developer working inside an existing Replit project.

Your task is to **perform broad, end-to-end user and features testing for the entire application** using ONLY what already exists in the project (frameworks, routing, components, etc.). Do NOT install any new external tools unless such a framework is already present in the repo.

---

## Overall Goal

Systematically review and verify that:

1. **All user flows work from start to finish** for every user type.
2. **All features, pages, buttons, and forms are active and functional**.
3. **CRUD flows (add/edit/update/delete)** exist and behave correctly.
4. **Navigation and UX are fully operational.**
5. **Permissions and role-based access** work correctly.
6. **No logic gaps** exist in how users move through the app.
7. **Design consistency** is maintained across all pages and components.

Use ONLY the existing code, patterns, and UI.

## **Step 8 – DESIGN CONSISTENCY TESTING** *(Added Section)*

Perform a **complete design consistency audit** across all pages and components. Ensure:

### 1. Global Layout Consistency
- Header/nav appears the same across all pages  
- Logo placement is consistent  
- Footer is consistent  
- Page widths and responsive breakpoints behave predictably  
- No pages use outdated or mismatched layouts  

### 2. Typography Consistency
- Heading hierarchy (H1/H2/H3) is consistent  
- Font sizes, weights, and line heights are uniform  
- Body text uses consistent classes/styles  
- Avoid mismatched old styles vs. new styles

### 3. Component Consistency
Check consistent usage of:
- Buttons (primary/secondary/destructive)
- Cards, tables, lists
- Form inputs (text fields, dropdowns, toggles)
- Modals and side panels
- Alerts, notifications, and banners

Verify:
- Spacing, padding, and margin systems are consistent  
- Components use the same design tokens or classes  
- No visual drift across routes  

### 4. Color & Branding Consistency
- Colors match the style guide / brand palette  
- No off-brand colors in legacy pages  
- Backgrounds and section dividers are consistent  
- Dark/light theme behavior is stable (if applicable)

### 5. Iconography & Imagery
- Icon style and weight is consistent  
- Images and illustrations follow a unified aesthetic  
- No pixelated, misaligned, or stretched assets  

### 6. Responsive Design
Test each page at:
- Mobile
- Tablet
- Desktop

Ensure:
- Layouts adapt correctly  
- Buttons/inputs remain usable  
- Nothing overlaps or renders off-screen  
- No scroll traps or oversized fixed elements  

### 7. Alignment & Spacing Check
Perform a pass to verify:
- Consistent spacing scale  
- Consistent alignment of text, cards, buttons, and forms  
- No random offsets or mismatched padding  

Document any inconsistent patterns in `DESIGN_ISSUES.md`.
